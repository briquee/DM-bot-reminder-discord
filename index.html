RappelBot — Rappels en DM

Bot Discord simple pour programmer des rappels envoyés en message privé (DM). Ci-dessous : guide d'utilisation, commandes, Terms of Service et politique de confidentialité prêtes à l'emploi.

Voir les commandes Terms Privacy

Fonctions

Rappels en DM

Envoie des rappels personnels à l'utilisateur via DM à l'heure programmée.

Commandes simples

Créer, lister, annuler des rappels via des commandes slash ou en message.

Persistant

Les rappels sont sauvegardés côté serveur (base de données) avec une période de rétention configurable.

Commandes

Exemples de commandes (slash commands recommandées) — adaptez selon ton code.

/remindme add 2025-11-05 09:00 "Appeler le client"
/remindme list
/remindme cancel 42

Exemple minimal (pseudo-code) :

POST /api/reminders
{ "user_id": "1234567890", "time": "2025-11-05T09:00:00Z", "text": "Appeler le client" }

Le responsable du traitement des données est le propriétaire du bot (contact : brique1009@gmail.com).

2. Données collectées

Nous collectons et traitons les données minimales nécessaires pour fournir le service de rappels :

  • ID Discord (user id) — pour identifier le destinataire du rappel.
  • Contenu du rappel (texte) — la note que vous demandez d'envoyer.
  • Horodatage — date et heure du rappel.
  • Métadonnées — logs d'exécution, statut du rappel, éventuellement des informations techniques (IP/agent serveur) si pertinentes pour la sécurité.

3. Finalités du traitement

Les données sont utilisées pour :

  • Créer et envoyer des rappels à l'utilisateur sur Discord.

Les rappels sont conservés tant que nécessaire pour l'exécution du service. Les logs d'audit et les métadonnées peuvent être conservés jusqu'à 90 jours par défaut (paramétrable).
